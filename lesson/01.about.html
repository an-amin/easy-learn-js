What is JavaScript? (জাভাস্ক্রিপ্ট কি? )

JavaScript is an object-oriented computer programming language commonly used to create interactive effects within web browsers.

জাভাস্ক্রিপ্ট একটি কম্পিউটার প্রোগ্রামিং ভাষা যা ওয়েব ব্রাউজারে প্রাণবন্ত সব ইফেক্ট/ক্রিয়া-প্রতিক্রিয়া তৈরি করতে ব্যবহার করা হয়।

JavaScript ("JS" for short) is a full-fledged dynamic programming language that, when applied to an HTML document, can provide dynamic interactivity on websites. It was invented by Brendan Eich, co-founder of the Mozilla project, the Mozilla Foundation, and the Mozilla Corporation.

জাভাস্ক্রিপ্ট (সংক্ষেপে JS) একটি পরিপূর্ণ, গতিশীল (ডাইনামিক), পোগ্রামিং ভাষা, যার সাহায্যে HTML ডকুমেন্টে প্রয়োগ করে সুন্দর এবং প্রয়োজনীয় সব গতিশীল ইফেক্ট/ ক্রিয়া-প্রতিক্রিয়া এবং HTML এলিমেন্টসমুহের মধ্যে মিথস্ক্রিয়া রচনা করা যায়, যা ওয়েব পেইজকে আরও প্রাণবন্ত করে তোলে। মজিলা ফাউন্ডেশন এবং কর্পোরেশনের সহপ্রতিষ্ঠাতা জনাব ব্রেনডান আইচ এর রচয়িতা।

Java & JavaScript! (জাভা এবং জাভাস্ক্রিপ্ট!)

Do not confuse JavaScript with the Java programming language. Both Java and JavaScript are trademarks or registered trademarks of Oracle in the U.S. However, the two programming languages have very different syntax, semantics, and uses.

JavaScript এবং Java programming language সম্পূর্ণ ভিন্ন দুইটি প্রোগ্রামিং ভাষা। Java এবং JavaScript দুটি ভাষাই যুক্তরাষ্ট্রের Oracle কোম্পানি কর্তৃক ট্রেডমার্ক নিবন্ধিত। দুটি ভাষারই syntax, semantics এবং ব্যবহার ভিন্ন।

Use of JavaScript (জাভাস্ক্রিপ্ট-এর ব্যবহার)

JavaScript is an important programing language in web technology.

it is used...
  • to change HTML & CSS
  • HTML এবং CSS পরিবর্তনে
  • to make dynamic & interactive webpage
  • গতিশীল এবং প্রাণবন্ত ওয়েব পেইজ তৈরি করতে
  • to make image library, file-manager, carousel & slideshow etc...
  • ফাইল ম্যানাজার, ইমেজ লাইব্রেরি, ইমেজ স্লাইড, ইত্যাদি তৈরি করতে...
  • to make popups, modals, notifications & alerts etc...
  • পপ-আপ মোডাল,নোটিফিকেশন, এলার্ট ইত্যাদি তৈরি করতে...
  • to control events like on(click, double-click, mouseover-mouseouut, form-submit etc) and their actions...
  • অন: ক্লিক/ডাবলক্লিক, লোড/আনলোড, মাউসওভার/আউট, ফর্ম-সাবমিট ইত্যাদি HTML ইভেন্ট নিয়ন্ত্রণ এবং ক্রিয়াশীলতা/কার্যপ্রণালীতে সমন্বয় সাধনের জন্য
  • to make synchronize & asynchronize requests without reloading current webpage
  • ওয়েব পেইজ রিলোড না করেই / সিনক্রোনাইজ/ এসিনক্রোনাইজ রিকুয়েস্ট পাঠানো এবং এর মাধ্যমে প্রাপ্ত তথ্য নিয়ে কাজ করতে।
  • as API platform to communicate among different applications.
  • এপ্লিকেশনসমূহের মধ্যে যোগাযোগের জন্য API প্লাটফর্ম হিসেবে

JavaScript Libraries (জাভাস্ক্রিপ্ট লাইব্রেরি)

A JavaScript library is a library of pre-written JavaScript codes/function which allows to re-use codes and handles different issues for easier development.

জাভাস্ক্রিপ্ট লাইব্রেরি হচ্ছে জাভাস্ক্রিপ্ট-এ লিখা কোড/ফাংশন যা কোনও নির্দিষ্ট সমস্যা সমাধান করে বা সমাধান সহজ করে।

collection of different types of libraries is called framework, works to solve more & complex problems...

একাধিক লাইব্রেরীর সমাহারকে ফ্রেমওয়ার্ক বলা হয় যা আরও বিবিধ এবং জটিল সমস্যা সমাধানে কাজ করতে পারে।

Advantages (সুবিধাসমূহ)
  • makes development easier & save time
  • এপ্লিকেশন তৈরি সহজ করে এবং সময় বাঁচায়
  • Library/Framework itself handles platform/error/security & other issues
  • লাইব্রেরি/ফ্রেমওয়ার্ক নিজেই প্লাটফর্ম/ভুলত্রুটি/ নিরাপত্তা ইত্যাদি বিভিন্ন বিষয় সামাল দেয়। এ নিয়ে ডেভেলপারকে অতিরিক্ত মাথা ঘামাতে হয় না।
  • code re-usibility allows better performance
  • কোড পূণঃব্যবহারের কারণে এপ্লিকেশনের কর্মদক্ষতা ভালো থাকে

কোর-জাভাস্ক্রিপ্ট নাকি জাভাস্ক্রিপ্ট-লাইব্রেরি/ফ্রেমওয়ার্ক ?

HTML, CSS & JS -ছিল, আছে এবং থাকবে... ওয়েব এপ্লিকেশন ডেভেলপমেন্ট ইন্ড্রাস্ট্রিতে এদের কোনও বিকল্প নাই। র‍্যাপিড ডেভেলপমেন্ট এর জন্য কোনও একটা লাইব্রেরি/ফ্রেমওয়ার্ক জানা থাকা অবশ্যই ভালো, দরকারও বটে, কারণ কোর-জাভাস্ক্রিপ্ট দিয়ে সমগ্র প্রজেক্ট শেষ করতে অনেক সময় লেগে যাবে, আর লাইব্রেরি সেই সময়ট এবং স্ট্রেস দুটোই অনেক কমিয়ে দেবে,

আরেকটা সাধারণ ব্যপার হচ্ছে, নির্দিষ্ট যে সমস্যাগুলোর সুন্দর এবং সহজ সমাধান তৈরি করাই আছে সেগুলো পুনরায় তৈরি করা প্রশ্নই আসে না! সুতরাং সেই সুযোগটা আমরা অবশ্যই নিবো এবং তার জন্য আমরা প্রচলিত লাইব্রেরিগুলোও স্টাডি করবো...

কিন্তু কথা হচ্ছে তার জন্য কোরঃ বাদ-দিয়ে দিলে বিপদ হবে। অন্ততঃপক্ষে ব্যাসিক জাভাস্ক্রিপ্ট শিখতে/জানতেই হবে, তা না হলে লাইব্রেরি সমূহও আমরা ঠিকমত বুঝতে/ব্যবহার করতে পারবো না, আর কোনও মতে টুকটাক ব্যবহার করতে পারলেও তার থেকে পূর্ণ সুবিধা নিতে পারবো না। নতুন কোনও সমস্যা/চ্যালেঞ্জ আসলে যা প্রচলিত লাইব্রেরি সমাধান করেনা ওইসব ক্ষেত্রে ঝামেলায় পড়বো... ইত্যাদি...
